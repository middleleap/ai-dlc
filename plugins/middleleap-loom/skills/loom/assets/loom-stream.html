    The controls

    Nineteen gates — run by a risk-scoped runner

    Each is pure-Node, tested, and blocks merge on failure — with a CI negative test showing an attempted bypass being rejected. The gate runner reads the control catalog's lanes (pr · release · scheduled) and path scopes, so a documentation fix does not take a lending model's route — and every skipped gate is recorded with its reason, never silent.

    Honest self-grade

    How bank-grade it is

    The Loom grades itself: a bundle ships seams, declaration-gates, agents, and runbooks — it can't ship real IAM, an independent function, external WORM, real-PII crypto, SRE, or a regulator's opinion.

    ~30 mechanically validated ~17 defined ~12 absent

    The rule that keeps it honest: a gate that validates a declaration is mechanically validated, never "enforced" — the word enforced is earned by the platform control plus a negative test showing an attempted bypass being rejected. As shipped: platform enforced 0 · organisationally enforced 0; those states are the adopter's to claim, per control, with activation evidence.

    The five states (1.10): Absent · Defined · Mechanically validated · Platform enforced · Organisationally enforced. The state of record is the machine-readable control-catalog.json, checked by its own gate — an entry claiming a state without its receipts fails the build. This traffic-light is the narrative view; the catalog wins.
