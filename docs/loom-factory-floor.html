Git is the loom. Notion is the factory floor.

The Loom already produces every artifact a team needs to collaborate around — passports, envelopes, hand-offs, a backlog. What it never shipped is the floor: the shared surface where humans and agents see the work, discuss it, and decide. Notion has just built exactly that seam. This page shows how the two connect — without a single control moving out of git.

The idea

The software factory — the coordination system, not the coder.

The term has two lives. Both end at the same conclusion: the factory is the system around the workers, and someone has to build its floor.

1968 →

The classic factory

Industrial principles applied to software: standardised process, reusable components, quality checkpoints. NATO 1968 named it; NEC, Hitachi and Fujitsu built it. Assemble from parts — hand-code only what is new.

2025–26

The AI-native reframing

The coding agent is the unit; the factory is the interconnected system that triages, plans, builds, reviews and ships through coordinated agents — with humans designing and overseeing the system itself.

Notion · Token Town

The floor as a database

Notion builds Notion this way: 100+ tools, 30+ internal agents, and one coordination substrate — agents write tasks to a database other agents listen to. Humans design the system; the workspace is the floor.

The resonance is exact. The Loom's governing principle — AI proposes; humans and a protected control plane dispose — is the same sentence in a regulator's register. The Loom is the machinery and the controls. It never named the floor. Notion did.

The teammates

Two kinds of teammates — humans on the floor, agents in the queue.

The same surface serves both. Each dimension carries one hard line that never moves.

Human collaboration

  • A board over the backlog — swimlanes by role, "what's waiting on me"
  • PA1/PA2 routed to all 12 roles; who is blocking is visible
  • Threads bound to the passport, the change, the hand-off
  • Discovery co-authored; D9 reactions in place, exported by PR
  • A standing MI view over the sealed evidence
The hard lineThe approval page carries the evidence in — change summary, gate results, decision-log excerpt. Routing must increase reading, not just throughput.

Agent orchestration

  • The delivery queue mirrored to a database agents listen to
  • Webhooks and Workers as assembly-line events
  • Claude Code works the floor as a named partner agent
  • Teammates propose work from the floor — by PR, into the backlog
  • Humans stay at the merge and launch gates, always
The hard lineThe waist gate is never bypassed. A card is not a hand-off; the authoritative queue stays docs/backlog.yaml, and feature entry still requires a gate-green handoff.md.

The appearance-of-control trap

A board can look like governance while the controls sit elsewhere. Git is truth; decisions round-trip as signed evidence; nothing is green on the floor's say-so.

Comprehension-debt acceleration

Four-eyes can decay into ceremony while every gate stays green — and a one-click approval makes rubber-stamping easier. The approval page must carry the evidence in, so routing increases reading.

The system-of-record temptation

The moment a team edits truth on the floor first, the evidence chain breaks. The projection stays a projection, with an explicit PR round-trip.

Data residency & PII — HG-0011

Regulated governance state leaving git for a SaaS workspace is a real control question. Project references and status, never personal data; the residency review gates adoption, it does not follow it.

A moving platform

The 2025-09-03 API split databases into data sources — a breaking change. The hosted MCP is not headless; the self-hosted one may sunset. Pin to the REST API and re-verify before building.

Cost and tier

Custom Agents and connectors are Business/Enterprise only, priced in credits — a per-adopter cost, not free machinery.
