## Each Entry Contains

- **Use Case Name** — Short, catchy title with journey type in parentheses
- **Journey Type(s)** — SIP, FDP, FD-MULTI, VRP, BULK, INTL, DATA, or COMBINED
- **Channel Context** — Deployment environment (E-Commerce, Fintech, Bank, Government, etc.)
- **Problem Statement** — 1-2 sentences on the pain point
- **Value Proposition** — 2-3 sentences on the solution
- **Key Benefits** — 3-4 bullet points for customer, business, bank, ecosystem
- **Sample Scenario** — Realistic UAE scenario with real brand names (Noon, DEWA, du, ADCB, Emirates NBD, FAB, Careem, etc.)
- **Presentation Headline** — One-liner for slide title
- **KPIs** — 2-3 success metrics

## How to Use for Rapid Prototyping

### Generate a Pitch Slide (5 min)
1. Pick a use case entry
2. Use **Presentation Headline** as slide title
3. Add **Problem Statement** + **Value Proposition** as narrative
4. Pull 2-3 **Key Benefits** as bullet points
5. Include **KPIs** on a success metrics slide

### Create a User Flow (15 min)
1. Select an entry and note its **Journey Type(s)**
2. Map the flow: user action → bank auth → payment/data → confirm
3. Use **Sample Scenario** to create realistic personas
4. Build wireframes: initiation → redirect → authorization → receipt

### Prepare for Stakeholder Meeting (10 min)
1. Lead with **Presentation Headline**
2. Use **Sample Scenario** for concrete examples
3. Map **Key Benefits** to each stakeholder (customer, merchant, bank, ecosystem)
4. Close with **KPIs** to align on success

## Journey Types Quick Reference

| Type | Full Name | Use | Characteristics |
|------|-----------|-----|-----------------|
| **SIP** | Single Immediate Payment | One-off payment | Instant, customer-initiated |
| **FDP** | Fixed Duration Payments | Scheduled payments | Fixed amount, fixed schedule (e.g., 3 months) |
| **FD-MULTI** | Fixed Duration Multi-Payment | Installments | Variable # of fixed amounts; scheduled |
| **VRP** | Variable Recurring Payments | Recurring, flexible | Variable amounts; auto-executes within limits |
| **BULK** | Bulk / Batch Payments | B2B payroll | Multiple payments in one instruction |
| **INTL** | International Payment | Cross-border | Multi-currency, international settlement |
| **DATA** | Data Sharing / AIS | Account information | 6-12 month transaction history access |
| **COMBINED** | Multi-Journey | Complex scenarios | Combines 2+ journey types |

## Tips for Maximum Effectiveness

1. **Tailor the Scenario** — Replace brand names with your specific partner (e.g., your bank instead of ADCB)
2. **Adjust KPIs** — Use these as templates; adapt metrics to your business model
3. **Combine Entries** — Mix two use cases for novel applications (e.g., E-Commerce + BNPL)
4. **Challenge Assumptions** — Use Problem Statements to identify new markets or gaps
5. **Create Variants** — The same journey type (e.g., SIP) works for many use cases; create vertical-specific versions
